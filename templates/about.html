How the model works

We preprocess inputs (date → quarter, one-hot encode categorical fields, align to training features) and use a Random Forest regressor to predict a productivity score (0–1). The score is mapped to categories: Low / Medium / High for easier interpretation.

Key pipeline steps

  1. Input validation & date parsing
  2. Feature encoding & alignment with model features
  3. Model prediction (Random Forest)
  4. Category mapping + interpretation + recommended actions

Data & privacy

Model trained on anonymized factory logs. No personal identifiers are stored or exported by default. If you provide sample data through the form, it's used only for real-time prediction and not persisted unless you explicitly download or save results.

Privacy highlights

  • Inputs are processed in-memory for prediction
  • Downloads (CSV) are generated on demand only
  • No automatic telemetry or user tracking
